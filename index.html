Introduction

Welcome to the API documentation. This API provides various endpoints for monitoring and status checking of services.

Base URL: https://api.example.xd

Authentication

API authentication may be required depending on the endpoint. Please contact your administrator for details.

Health Check Endpoints

GET /api/health

Returns the health status of the MongoDB service.

Example Request

curl -X GET 'https://api.example.xd/api/health'

Example Response

{
  "status": "MongoDB is online!",
  "lastChecked": "Mar 01, 2025, 06:17:51 PM",
  "connectionTime": "2025-03-01T23:17:51.958Z",
  "retryCount": 0
}
GET /api/redishealth

Returns the health status of the Redis service.

Example Request

curl -X GET 'https://api.example.xd/api/redishealth'

Example Response

{
  "status": "Redis is online!",
  "lastChecked": "Mar 01, 2025, 06:17:29 PM",
  "connectionTime": "2025-03-01T23:17:29.571Z",
  "retryCount": 0
}

Service Handler Endpoints

GET /api/sh1

Service handler 1 endpoint for retrieving shard information.

Example Request

curl -X GET 'https://api.example.xds/api/sh1'

Example Response

{
  "shardId": "0",
  "guildCount": 7,
  "memberCount": 1367,
  "ping": 84,
  "uptime": 2822951,
  "timestamp": "2025-03-01T23:17:38.255Z"
}
GET /api/sh2

Service handler 2 endpoint for retrieving shard information.

Example Request

curl -X GET 'https://api.example.xd/api/sh2'

Example Response

{
  "shardId": "1",
  "guildCount": 2,
  "memberCount": 16,
  "ping": 89,
  "uptime": 2823074,
  "timestamp": "2025-03-01T23:17:46.053Z"
}

Response Fields Explanation

Health Check Response Fields

Field Type Description
status String Status message indicating if the service is online
lastChecked String Human-readable timestamp of when the service was last checked
connectionTime String (ISO 8601) ISO formatted timestamp of the connection time
retryCount Integer Number of connection retry attempts

Shard Handler Response Fields

Field Type Description
shardId String Identifier for the shard
guildCount Integer Number of guilds managed by this shard
memberCount Integer Number of members across all guilds in this shard
ping Integer Latency measurement in milliseconds
uptime Integer Uptime in milliseconds
timestamp String (ISO 8601) ISO formatted timestamp of when the data was collected

Error Codes

The API uses the following error codes:

Status Code Description
200 OK - The request was successful.
400 Bad Request - The request was invalid.
404 Not Found - The endpoint does not exist.
500 Internal Server Error - An error occurred on the server.
503 Service Unavailable - The service is not available.

Error Response Format

{
  "error": "Error message describing the issue",
  "status": 500,
  "timestamp": "2025-03-01T23:17:51.958Z"
}
